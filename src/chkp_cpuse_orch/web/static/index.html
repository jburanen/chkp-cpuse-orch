Check Point Patch Management

Check Point CPUSE and CDT patch orchestration

Provision management server access

Generates the clish commands that create this tool's service account on a management server — paste them into clish on each server. The account's shell is /bin/bash so SCP package staging works (the tool wraps its CPUSE commands in clish -c to match). The password appears in the output only as a salted SHA-512 hash. Afterwards, save the same username/password in the Packages and Credentials tab.

Patch management servers

      

        Patched locally via CPUSE (CDT does not patch management servers).
        Refresh queries live show installer packages state over SSH.
        Import uploads the selected package and imports it into the local
        CPUSE repository. Install appears next to imported packages —
        it can reboot the server, so it always asks for confirmation.

Gateway deployment (CDT)

      

        CDT runs on a management server and pushes a package to many gateways.
        Flow: Stage (upload package + write CDT config) →
        Generate candidates → Load, reorder and trim the
        list (row order = deployment order = blast-radius control) → Save order
        → optional Prepare → Execute. Execute runs under
        nohup on the server and is followed live in the Jobs section.

New deployment

      
    

    

      This looks like a fresh installation: one empty environment named
      default and no data yet. Environments are independent management
      estates (own servers, own credentials) selected with the top-right picker.
      If you'd like a more meaningful name — a site, a customer, a datacenter —
      now is the easiest time, before servers and credentials are attached to it.
    

    

      
      
      
    

  








  

    

      
New environment

      
    

    

      Environments are independent management estates — each has its own inventory
      of management servers and its own credentials. After creating one, add its
      servers on the Provisioning tab and its credentials on
      Packages and Credentials.
