Bootstrap initial management access

Generates clish commands to provision this tool's service account on management servers. These credentials are saved to the Credentials table below automatically; edit that entry to add the API key once you generate one.

Gateway deployment (CDT)

      

        CDT runs on a management server and pushes a package to many gateways.
        Flow: Stage (upload package + write CDT config) →
        Generate candidates → Load, reorder and trim the
        list (row order = deployment order = blast-radius control) → Save order
        → optional Prepare → Execute. Execute runs under
        nohup on the server and is followed live on the Jobs tab.

New deployment

      
    

    

      This looks like a new deployment, would you like to rename the default environment? 
    

    

      
      
      
    

    
 
      Environments are analogous to your Check Point management systems. Each related 
      group of management servers should have an environment defined here. That means any 
      primary management server and related SmartEvent and backup/HA servers. If you are 
      connecting to Multi-Domain, you should create one environment for the whole MDS 
      system, not one per domain.
    

  








  

    

      
Manage environments

      
    

    

      Environments are independent management estates — each has its own inventory
      of management servers and its own credentials. Edit a name and click
      Rename: servers, credentials, and job history move with it.
      Servers and deletion are managed on the Provisioning tab. An
      environment is always entirely an SMS estate or entirely a Multi-Domain
      (MDS) one — never a mix — so discovery and other commands know which
      variant to use.
